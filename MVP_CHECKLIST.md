# GateForge MVP Checklist

Use this checklist before tagging and packaging MVP.

## A. Repo And CI

- [ ] `git status` is clean (no pending tracked changes).
- [ ] Main CI is green on the target commit (`test-and-smoke` must pass).
- [ ] Optional benchmark/governance/freeze jobs are green for at least one workflow_dispatch run.

## B. Core Runtime Path

- [ ] Proposal-driven run works:

```bash
python3 -m gateforge.run \
  --proposal examples/proposals/proposal_v0.json \
  --baseline auto \
  --out artifacts/mvp_check/proposal_run.json
```

- [ ] Output contains status and policy decision:
  - `artifacts/mvp_check/proposal_run.json`
  - `artifacts/mvp_check/proposal_run.md` (if generated by current config)

## C. Medium Governance Path

- [ ] Medium benchmark dashboard chain passes:

```bash
bash scripts/demo_medium_pack_v1_dashboard.sh
```

- [ ] Dashboard artifact exists and has `bundle_status`:
  - `artifacts/benchmark_medium_v1/dashboard.json`

## D. Freeze Gate

- [ ] MVP freeze check passes:

```bash
bash scripts/mvp_freeze_check.sh
```

- [ ] Freeze verdict is pass in:
  - `artifacts/mvp_freeze/summary.json`
  - expected `verdict`: `MVP_FREEZE_PASS`

## E. Agent-In-Loop Sanity

- [ ] Autopilot dry-run still works:

```bash
bash scripts/demo_autopilot_dry_run.sh
```

- [ ] Agent change loop demo still matches expected statuses:

```bash
bash scripts/demo_agent_change_loop.sh
```

## F. Release Hygiene

- [ ] `README.md` and `DEMO.md` are consistent with actual commands/artifact paths.
- [ ] `.env` is not committed; `.env.example` is up to date.
- [ ] Create release tag only after all above checks are done.
