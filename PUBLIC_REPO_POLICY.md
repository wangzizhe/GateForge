# Public Repository Policy

This repository contains reusable GateForge software infrastructure.

Public commits may include:

- Source code for reusable runtime components.
- Tests based on minimal synthetic fixtures.
- CI configuration.
- High-level project documentation.

Public commits must not include:

- Local credentials or environment files.
- Non-public research assets or notes.
- Generated run artifacts.
- Non-public data files.
- Internal decision logs, roadmaps, or experiment ledgers.

Keep non-public material in ignored local paths or in a separate repository
with restricted access.
